# Science Review Checklist

Final scientific review before accepting, publishing, presenting, or using a result.

## Questions
1. Reproducible?  2. Producing script known?  3. Scenario config known?
4. Output artifact known?  5. Source commit known / recoverable?  6. Comparison fair?
7. Assumptions documented?  8. Truth-model level documented?  9. Geometry equivalent
across compared cases?  10. Visibility equivalent?  11. Measurement count equivalent?
12. Station coverage equivalent?  13. Residuals physically reasonable?  14. Residuals
biased?  15. Residuals trending?  16. Residuals autocorrelated?  17. Covariance
credible?  18. NIS / NEES reasonable?  19. Observability adequate?  20. Sensitive to
station geometry?  21. Sensitive to measurement diversity?  22. Improvement
statistically significant?  23. Would the conclusion survive Monte Carlo?  24. Is
there a simpler explanation for the improvement?  25. Traceable script -> config ->
artifact -> commit?

## Conclusion label (pick one)
Accept · Accept with caveats · Needs rerun · Needs baseline comparison · Needs
Monte Carlo support · Needs external cross-validation · Reject as unsupported

## Critical rule
Do not present a result as scientifically strong if it rests only on a single run,
unknown configuration, unknown seed, or untraceable artifact.
